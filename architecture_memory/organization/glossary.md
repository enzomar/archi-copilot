# Enterprise Architecture Glossary

> Common terminology used across all projects

## Architecture & Governance Terms

| Term | Definition |
|------|------------|
| **ADR** | Architecture Decision Record - documented architecture decision |
| **CAB** | Change Advisory Board - approves production changes |
| **NFR** | Non-Functional Requirement - quality attributes |
| **SLA** | Service Level Agreement - contracted service commitments |
| **SLO** | Service Level Objective - internal service targets |
| **TOGAF** | The Open Group Architecture Framework |
| **RFC** | Request for Comments - proposal document |

## Development & Operations Terms

| Term | Definition |
|------|------------|
| **CI/CD** | Continuous Integration / Continuous Deployment |
| **IaC** | Infrastructure as Code |
| **SRE** | Site Reliability Engineering |
| **MTTR** | Mean Time To Recovery |
| **MTBF** | Mean Time Between Failures |
| **RPO** | Recovery Point Objective - acceptable data loss |
| **RTO** | Recovery Time Objective - acceptable downtime |

## Technical Terms

| Term | Definition |
|------|------------|
| **ADR** | Architecture Decision Record - documented decision with context |
| **SLA** | Service Level Agreement - contractual performance commitment |
| **SLO** | Service Level Objective - internal performance target |
| **MTTR** | Mean Time to Recovery - average incident resolution time |
| **Strangler Fig** | Pattern for gradually replacing legacy systems |
| **TOGAF** | The Open Group Architecture Framework |
| **ADM** | Architecture Development Method (TOGAF cycle) |
